# Haru Live2D 模型目录结构完整说明

本文档详细说明 Haru Live2D 模型目录中每个文件和文件夹的作用，帮助开发者理解模型资源的组织结构。

---


## 📄 核心文件说明

### 1. `Haru.model3.json` - 主配置文件（入口文件）

**作用**：这是 Live2D 模型的**入口配置文件**，程序通过 `model.LoadModelJson()` 加载此文件。

**功能**：
- 定义模型版本（Version 3）
- 引用所有资源文件（MOC、纹理、物理、姿势、表情、动作等）
- 定义参数组（Groups）：眨眼组、口型同步组
- 定义点击区域（HitAreas）：头部、身体等可交互区域
- 组织动作分组（Motions）：Idle、TapBody 等

**在代码中的使用**：
```python
model.LoadModelJson("Resources/v3/Haru/Haru.model3.json")
# SDK 会自动解析此文件并加载所有引用的资源
```

**重要性**：⭐⭐⭐⭐⭐ 这是模型加载的入口，所有其他文件都通过此文件引用。

---

### 2. `Haru.moc3` - 模型核心数据文件

**作用**：包含模型的**核心结构数据**，是模型的基础。

**内容**：
- 网格（Mesh）数据：模型的顶点、面片信息
- 骨骼（Bone）数据：模型的骨骼结构
- 变形器（Deformer）数据：控制模型变形的参数
- 部件（Part）数据：模型的可控部件列表
- 参数（Parameter）定义：所有可控制的参数

**特点**：
- 二进制格式，由 Live2D Cubism Editor 导出
- 文件较大（通常几百 KB 到几 MB）
- 不可直接编辑，需要通过 Live2D 编辑器修改

**重要性**：⭐⭐⭐⭐⭐ 没有此文件，模型无法显示。

---

### 3. `Haru.cdi3.json` - 显示信息配置文件

**作用**：定义模型的**显示相关元数据**。

**内容**（从文件结构推断）：
- 画布尺寸（Canvas Size）
- 画布原点位置（Canvas Origin）
- 像素密度（Pixels Per Unit）
- 参数列表和分组信息
- 参数的中文名称映射（如 "角度 X"、"左目　開閉" 等）

**功能**：
- 告诉程序如何正确显示模型
- 提供参数的中文名称，方便调试和开发
- 定义参数的分组关系

**在代码中的使用**：
```python
# 获取画布信息
canvas_size = model.GetCanvasSize()
canvas_pixels = model.GetCanvasSizePixel()
pixels_per_unit = model.GetPixelsPerUnit()
```

**重要性**：⭐⭐⭐⭐ 影响模型的显示效果和参数访问。

---

### 4. `Haru.physics3.json` - 物理模拟配置文件

**作用**：定义模型的**物理模拟参数**，让头发、衣服等部件产生自然的晃动效果。

**内容**（从文件结构推断）：
- 物理设置组（PhysicsSettings）：
  - **前髪**（前发）：受头部角度影响
  - **横髪**（侧发）：受头部角度影响
  - **後ろ髪**（后发）：受头部角度影响
  - **スカーフ**（围巾）：受身体运动影响
- 每个物理设置包含：
  - 输入参数（Input）：如 `ParamAngleX`、`ParamAngleY` 等
  - 输出部件（Output）：受物理影响的部件
  - 物理参数：重力、阻力、弹性等

**效果**：
- 当模型头部转动时，头发会自然摆动
- 当模型身体移动时，围巾会跟随晃动
- 增加模型的真实感和生动性

**重要性**：⭐⭐⭐⭐ 让模型动画更自然，但非必需（可以关闭物理模拟）。

---

### 5. `Haru.pose3.json` - 姿势联动配置文件

**作用**：定义模型**部件之间的联动关系**。

**内容**（从文件结构推断）：
- 联动组（Groups）：
  - 右臂联动组：`Part01ArmRA001` 和 `Part01ArmRB001` 联动
  - 左臂联动组：`Part01ArmLA001` 和 `Part01ArmLB001` 联动

**功能**：
- 当移动右臂时，右臂的上下部分会协调运动
- 当移动左臂时，左臂的上下部分会协调运动
- 确保模型动作的协调性和自然性

**重要性**：⭐⭐⭐ 提升动作协调性，但非必需。

---

### 6. `Haru.userdata3.json` - 用户自定义数据文件

**作用**：存储模型的**自定义元数据**，用于扩展功能。

**内容**（从文件结构推断）：
- 用户数据列表（UserData）：
  - 为特定的 ArtMesh（艺术网格）添加标签
  - 例如：`D_PSD_27`、`D_PSD_25`、`D_PSD_24` 标记为 "tai"

**用途**：
- 标记特定部件（如用于点击检测）
- 存储自定义标签或元数据
- 扩展程序功能（如根据标签实现特殊效果）

**重要性**：⭐⭐ 可选功能，用于扩展。

---

## 📁 文件夹说明

### 1. `Haru.2048/` - 纹理贴图文件夹

**作用**：存储模型的**外观纹理图片**。

**文件**：
- `texture_00.png`：第一张纹理贴图（2048x2048 像素）
- `texture_01.png`：第二张纹理贴图（2048x2048 像素）

**说明**：
- `2048` 表示纹理分辨率为 2048x2048 像素
- 多个纹理可以用于：
  - 不同部件的贴图
  - 实现换装功能
  - 优化内存使用（按需加载）

**内容**：
- 模型的皮肤、衣服、头发等所有可见部分的图片
- PNG 格式，支持透明通道

**重要性**：⭐⭐⭐⭐⭐ 没有纹理，模型无法显示外观。

---

### 2. `expressions/` - 表情文件夹

**作用**：存储模型的**表情定义文件**。

**文件列表**：
- `F01.exp3.json` 到 `F08.exp3.json`（共 8 个表情）

**文件结构**（以 F01 为例）：
```json
{
    "Type": "Live2D Expression",
    "Parameters": [
        {
            "Id": "ParamMouthForm",
            "Value": 0.27,
            "Blend": "Add"
        }
    ]
}
```

**内容**：
- 每个表情文件定义一组参数值
- 参数包括：眼睛大小、嘴角位置、眉毛形状等
- `Value`：参数的目标值
- `Blend`：混合模式（Add=叠加，Multiply=相乘，Override=覆盖）

**在代码中的使用**：
```python
# 设置指定表情
model.SetExpression("F01")

# 随机选择表情
model.SetRandomExpression()

# 重置表情
model.ResetExpression()
```

**重要性**：⭐⭐⭐⭐ 丰富模型的表情表现。

---

### 3. `motions/` - 动作文件夹

**作用**：存储模型的**动作动画文件**。

**文件列表**（共 27 个动作文件）：
- `haru_g_idle.motion3.json`：待机动作
- `haru_g_m01.motion3.json` 到 `haru_g_m26.motion3.json`：各种动作

**文件结构**（简化示例）：
```json
{
    "Version": 3,
    "Meta": {
        "Duration": 4.97,      // 动作时长（秒）
        "Fps": 30.0,            // 帧率
        "Loop": true            // 是否循环
    },
    "Curves": [
        // 参数曲线数组，定义动作过程中各参数的变化
        {
            "Target": "Parameter",
            "Id": "ParamAngleX",
            "Segments": [/* 关键帧数据 */]
        }
    ]
}
```

**内容**：
- 动作时长和帧率信息
- 参数曲线（Curves）：定义动作过程中各个参数的变化
  - 角度参数（ParamAngleX/Y/Z）
  - 表情参数（ParamEyeLOpen、ParamMouthOpenY 等）
  - 部件透明度、颜色等

**动作分组**（在 `model3.json` 中定义）：
- **Idle 组**：待机动作（2 个动作）
  - `haru_g_idle.motion3.json`
  - `haru_g_m15.motion3.json`
- **TapBody 组**：点击身体触发的动作（4 个动作）
  - `haru_g_m26.motion3.json`（说话动作）
  - `haru_g_m06.motion3.json`（信息介绍动作）
  - `haru_g_m20.motion3.json`（日常动作）
  - `haru_g_m09.motion3.json`（信息动作）

**在代码中的使用**：
```python
# 播放指定动作组的随机动作
model.StartRandomMotion("Idle", priority=300)
model.StartRandomMotion("TapBody", priority=3)

# 播放指定动作组的指定动作
model.StartMotion("TapBody", 0, priority=3)  # 播放 TapBody 组的第 0 个动作
```

**重要性**：⭐⭐⭐⭐⭐ 让模型"动起来"，是交互的核心。

## 🔗 文件之间的关联关系

### 加载流程

```
1. 程序调用 model.LoadModelJson("Haru.model3.json")
   ↓
2. SDK 解析 model3.json，获取所有资源引用
   ↓
3. 自动加载以下文件：
   ├── Haru.moc3（核心数据）
   ├── Haru.2048/texture_00.png、texture_01.png（纹理）
   ├── Haru.cdi3.json（显示信息）
   ├── Haru.physics3.json（物理配置）
   ├── Haru.pose3.json（姿势配置）
   ├── Haru.userdata3.json（用户数据）
   ├── expressions/*.exp3.json（所有表情）
   └── motions/*.motion3.json（所有动作，按需加载）
```

### 运行时使用

```
用户交互
   ↓
程序调用 model.StartRandomMotion("TapBody")
   ↓
SDK 从 model3.json 中找到 TapBody 组
   ↓
随机选择一个动作文件（如 haru_g_m26.motion3.json）
   ↓
加载并播放动作
   ↓
如果有 Sound 字段，触发音频播放回调
```

## 🛠️ 开发建议

### 1. 修改模型外观
- 编辑 `Haru.2048/texture_*.png` 文件
- 保持文件名和路径不变
- 保持图片尺寸一致

### 2. 添加新表情
- 在 `expressions/` 目录添加新的 `.exp3.json` 文件
- 在 `Haru.model3.json` 的 `Expressions` 数组中添加引用

### 3. 添加新动作
- 在 `motions/` 目录添加新的 `.motion3.json` 文件
- 在 `Haru.model3.json` 的 `Motions` 对象中添加引用
- 可以创建新的动作组，或添加到现有组

### 4. 调整物理效果
- 编辑 `Haru.physics3.json`
- 修改重力、阻力、弹性等参数
- 注意：需要 Live2D Cubism Editor 来正确编辑

### 5. 调试参数
- 查看 `Haru.cdi3.json` 了解所有可用参数
- 使用 `model.GetParameterCount()` 和 `model.GetParameter(i)` 获取参数信息
- 使用 `model.SetParameterValue(id, value)` 控制参数

## 📝 总结

Haru Live2D 模型目录包含了一个完整的 Live2D 模型所需的所有资源：

- **核心文件**：model3.json（配置）、moc3（数据）、纹理（外观）
- **动画资源**：motions（动作）、expressions（表情）
- **增强功能**：physics（物理）、pose（联动）、cdi（显示信息）
- **扩展数据**：userdata（自定义数据）

所有文件通过 `Haru.model3.json` 统一管理和引用，程序只需加载这一个入口文件，SDK 会自动处理所有资源的加载和管理。
